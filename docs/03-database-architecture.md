# 03 — Database Architecture: Quran Saya CMS

Database = **Supabase (PostgreSQL)** yang **dibagikan** dengan aplikasi. CMS
memakai **service-role key** (server-side). App memakai **anon key**.

> **Aturan emas**: `users` & `adzan_notification` adalah kontrak app yang live.
> Semua perubahan **idempoten** & **tidak mengubah RLS** tabel tersebut.

---

## 1. Peta Tabel

| Tabel | Pemilik/penulis | Peran |
|-------|-----------------|-------|
| `users` | ditulis **app**, dibaca CMS | Perangkat + progres belajar |
| `adzan_notification` | ditulis **app**, dibaca CMS | Preferensi adzan per user |
| `admin` | CMS | Akun admin (login) |
| `custom_reminders` | CMS | Pengingat harian terjadwal |
| `notification_logs` | CMS | Audit pengiriman notifikasi |

## 3. DDL Lengkap

### 3.1. `users`
Superset kolom yang ditulis app. Tambahan kolom via `add column if not exists`.

```sql
create table if not exists public.users (
  id uuid not null default gen_random_uuid(),
  created_at timestamptz not null default now(),
  device_id varchar null,
  device_name varchar null,
  token_firebase varchar null,
  is_reminder boolean null default false,
  version varchar null,
  platform varchar null,
  last_opened_at timestamptz null,
  last_read_ayat integer null,
  learning_name varchar null,
  learning_avatar integer null default 0,
  learning_level integer null default 1,
  learning_xp integer null default 0,
  learning_streak integer null default 0,
  learning_longest_streak integer null default 0,
  learning_lessons_done integer null default 0,
  learning_last_active_at timestamptz null,
  constraint users_pkey primary key (id)
);
```

**Indeks**
```sql
create index if not exists idx_users_is_reminder on public.users (is_reminder);
create index if not exists idx_users_device_id on public.users (device_id);
create index if not exists idx_users_last_opened on public.users (last_opened_at desc);
create index if not exists idx_users_learning_active on public.users (learning_last_active_at desc);
```

### 3.2. `adzan_notification`
```sql
create table if not exists public.adzan_notification (
  id bigint generated by default as identity not null,
  created_at timestamptz not null default now(),
  user_id uuid null,
  city_name varchar null,
  is_imsak boolean null default false,
  is_subuh boolean null default false,
  is_dzuhur boolean null default false,
  is_ashar boolean null default false,
  is_maghrib boolean null default false,
  is_isya boolean null default false,
  imsak_time varchar null,
  subuh_time varchar null,
  dzuhur_time varchar null,
  ashar_time varchar null,
  maghrib_time varchar null,
  isya_time varchar null,
  constraint adzan_notification_pkey primary key (id)
);
create index if not exists idx_adzan_notification_user_id on public.adzan_notification (user_id);
```

### 3.3. `admin`
```sql
create table if not exists public.admin (
  id uuid not null default gen_random_uuid(),
  created_at timestamptz not null default now(),
  email varchar not null,
  password_hash text not null,        -- bcrypt via extensions.crypt
  full_name varchar null,
  is_active boolean not null default true,
  last_login_at timestamptz null,
  constraint admin_pkey primary key (id),
  constraint admin_email_unique unique (email)
);
```

### 3.4. `custom_reminders`
```sql
create table if not exists public.custom_reminders (
  id uuid not null default gen_random_uuid(),
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  title text not null,
  body text not null,
  schedule_time varchar not null,     -- 'HH:MM'
  is_active boolean not null default true,
  sort_order int not null default 0,
  constraint custom_reminders_pkey primary key (id)
);
create index if not exists idx_custom_reminders_active
  on public.custom_reminders (is_active, schedule_time, sort_order);
```

### 3.5. `notification_logs`
```sql
create table if not exists public.notification_logs (
  id bigint generated by default as identity not null,
  created_at timestamptz not null default now(),
  user_id uuid null,
  source_type varchar null,           -- adzan | reminder | manual | winback
  category varchar null,              -- subuh | custom_reminder | broadcast | streak_at_risk | ...
  title text null,
  body text null,
  status varchar null default 'queued', -- queued | sent | failed | partial
  error_message text null,
  scheduled_time varchar null,        -- 'HH:MM'
  metadata jsonb null,
  dedupe_key text null,
  sent_at timestamptz null,
  constraint notification_logs_pkey primary key (id)
);
create index if not exists idx_notification_logs_created_at on public.notification_logs (created_at desc);
create index if not exists idx_notification_logs_status on public.notification_logs (status, source_type);
create unique index if not exists idx_notification_logs_dedupe_key
  on public.notification_logs (dedupe_key) where dedupe_key is not null;
```

## 4. Fungsi RPC — `admin_login`

Login diverifikasi di database (bcrypt), bukan di aplikasi, agar hash tak pernah
keluar.

```sql
create or replace function public.admin_login(email text, password text)
returns table (id uuid, email text, full_name text)
language sql security definer set search_path = public
as $$
  select a.id, a.email::text, a.full_name::text
  from public.admin a
  where lower(a.email) = lower(trim(admin_login.email))
    and a.is_active = true
    and a.password_hash = extensions.crypt(admin_login.password, a.password_hash)
  limit 1;
$$;
```

## 5. Kunci Dedupe (Anti-Kirim-Ganda)

`notification_logs.dedupe_key` unik (partial index `where not null`). Format:

| Sumber | Pola `dedupe_key` |
|--------|-------------------|
| Adzan | `adzan:{tanggal}:{HH:MM}:{prayerKey}:{userId}` |
| Reminder | `reminder:{tanggal}:{HH:MM}:{reminderId}:{userId}` |
| Win-back | `winback:{tanggal}:{userId}` (1×/hari/user) |
| Manual | *(null)* — broadcast boleh berulang, tak di-dedupe |

Cron memeriksa `dedupe_key` yang sudah ada; hanya kirim yang **baru** atau yang
**retry** (status `failed`/`queued`).

## 6. RLS (Row Level Security)

| Tabel | RLS | Alasan |
|-------|-----|--------|
| `users` | **biarkan seperti app live** (anon dapat insert/update/select) | App menulis via anon key; mengubah = merusak app |
| `adzan_notification` | idem | idem |
| `admin`, `custom_reminders`, `notification_logs` | akses via **service-role** (CMS server) | Tak pernah diakses browser/anon |

> Karena CMS memakai **service-role** (melewati RLS) dan hanya berjalan
> server-side di balik login admin, keamanan bertumpu pada: (1) key hanya di
> server, (2) gerbang sesi admin, (3) `.env` gitignored.

## 7. Strategi Migrasi

- Satu berkas **`supabase/schema.sql`** yang **idempoten**:
  - `create table if not exists` + `add column if not exists` untuk tabel
    kontrak app (aman di DB produksi).
  - `create table if not exists` untuk tabel CMS.
  - `create index if not exists`, `create or replace function`.
- Seed reminder default hanya bila tabel kosong (`where not exists`).
- **Membuat admin pertama** (dijalankan manual, sekali):
  ```sql
  insert into public.admin (email, password_hash, full_name)
  values ('admin@quransaya.id',
          extensions.crypt('PASSWORD_KUAT', extensions.gen_salt('bf')),
          'Admin');
  ```

## 8. Volume & Kinerja

- `users` ~ ribuan baris → gunakan `count(head:true)` untuk KPI (bukan fetch
  massal). Indeks pada `is_reminder`, `last_opened_at`, `learning_last_active_at`.
- `notification_logs` tumbuh cepat → indeks `created_at desc` & `status`.
  Roadmap: kebijakan retensi (arsip/hapus > 90 hari) bila perlu.
- Segmentasi push → satu query terfilter `.limit(20000)` (batas aman).

## 9. Data yang TIDAK Ada di Supabase

Untuk kejelasan: berikut **tidak** disinkron (murni lokal di perangkat), jadi
CMS **tidak** bisa melihatnya: bookmark, tafsir, jadwal sholat cache, jurnal
ibadah detail, tilawah harian, khatam detail, isi belajar per-lesson. CMS hanya
melihat **ringkasan** `learning_*` di `users`.
